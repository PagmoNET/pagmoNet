# 1.0 Release — Open Items

Working toward v1.0.0 across the monorepo (base `Pagmo.NET` / `PagmoNet4j` + companions
`Pagmo.NET.Ipopt` / `PagmoNet4j.ipopt`). The IPOPT deferred-load (dlopen) pivot has landed;
BUILD workflows are green and the RELEASE workflows were reworked to the payload model +
in-run reusable base build + fully-offline clean-room. See `../WORK_LOG.md` for the live status
of the release-workflow CI iteration. This file tracks what remains beyond that.

---

## Resolved: Structural — monorepo
The old duplicated-`pagmoNet`-submodule drift is gone: `pagmoNet/` is now a single monorepo
(no nested checkouts, no gitlinks to bump). The shared native/SWIG layer lives once under
`native/` + `swig/`.

---

## Open: Known API / Feature Gaps

Document these in READMEs / ROADMAP so users aren't surprised:

- **`free_form` topology** — dynamic edge add/remove not wrapped in Java (pagmo2 API exists)
- **Gradient/hessian callbacks** — not tested end-to-end in Java; low-use path
- **Kotlin wrappers for BFE / hypervolume / multi-objective utils** — use Java API directly for now
- **`namespace pagmo` in generated C#** — rename to `namespace PagmoNet` is a v2.0 breaking change; deferred
- **Runtime topology replacement** on managed islands — set topology at construction time
- **ARM CPU support** — not planned for v1.x

---

## Resolved: Licensing

Licensing is settled (see [LICENSING.md](../LICENSING.md)):
- **pagmo.NET, PagmoNet4j** (base) → MPL-2.0
- **pagmo.NET.ipopt, PagmoNet4j.ipopt** (companion) → EPL-2.0

Bundled third-party components keep their own licenses (pagmo2, NLopt, etc.); see each
package's NOTICE / THIRD_PARTY_LICENSES.md.
