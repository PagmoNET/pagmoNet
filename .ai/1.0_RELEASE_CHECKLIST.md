## Open: Structural

### Duplicated `pagmoNet` checkout drift
`pagmoNet` is checked out twice as a nested submodule (`pagmo.NET.ipopt/pagmo.NET/pagmoNet`
and `PagmoNet4j.ipopt/PagmoNet4j/pagmoNet`). Every shared native/SWIG change must be
mirrored into both working trees by hand, then gitlinks bumped up two parent chains.
This drifts in practice. Evaluate: `git subtree` for `pagmoNet`, or fold back into a monorepo.
**Post-1.0 design decision — needs discussion before starting.**

## Open: Known API / Feature Gaps

Document these in READMEs / ROADMAP so users aren't surprised:

- **`free_form` topology** — dynamic edge add/remove not wrapped in Java (pagmo2 API exists)
- **Gradient/hessian callbacks** — not tested end-to-end in Java; low-use path
- **Kotlin wrappers for BFE / hypervolume / multi-objective utils** — use Java API directly for now
- **`namespace pagmo` in generated C#** — rename to `namespace PagmoNet` is a v2.0 breaking change; deferred
- **Runtime topology replacement** on managed islands — set topology at construction time
- **ARM CPU support** — not planned for v1.x
